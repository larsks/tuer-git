Comment générer et lancer le jeu

  • Kit de développement Java 1.8 ou supérieur : ici, ici ou dans le dépôt officiel de votre distribution GNU Linux
  • Eclipse 4.3.2 ou supérieur (optionel mais recommandé): ici
  • Subversion (SVN)
    • utilisateurs en ligne de commande: ici
    • utilisateurs d'Eclipse: ici
  • ANT: ici (déjà supporté et livré avec les EDI majeurs tel qu'Eclipse et Netbeans)

Étapes de génération

Tapez ceci quand vous êtes dans le répertoire d'ANT pour télécharger les JARs requis par quelques têches optionnelles : ant networking -f fetch.xml -Ddest=system

Veuillez suivre une de ces méthodes :

  • En ligne de commande, suivez ces étapes :
    • tapez ceci pour obtenir le code source : svn checkout URL_du_dépôt_SVN
    • tapez ceci pour construire le projet : ant build
    • tapez ceci pour lancer le jeu : ant run-game
  • Dans Eclipse (version 4.3.2 ou supérieure), avec Subclipse (version 1.6.0 ou supérieure), suivez ces étapes :
    • Faites un clic droit dans la zone de paquetage
    • Sélectionnez "New" -> "Other..."
    • Ouvrez le répertoire "SVN"
    • Sélectionnez "Checkout Projects from SVN"
    • Cliquez sur "Next"
    • Sélectionnez "Create a new repository location"
    • Cliquez sur "Next"
    • Entrez l'URL du dépôt SVN dans le champ de texte "URL"
    • Cliquez sur "Next"
    • Sélectionnez le répertoire le plus haut
    • Cliquez sur "Finish"
    • Suivez ces étapes pour construire le projet avec Ant dans Eclipse :
      • Allez dans le menu "Window" -> "Show view" -> "Ant"
      • Glissez et déposez le fichier build.xml dans la vue ANT
      • Faites un doucle clic sur la cible "build" dans cette vue pour construire le projet
      • Faites un doucle clic sur la cible "run-game" dans cette vue pour lancer le jeu
  • Dans Netbeans (version 6.9.1 ou supérieure), suivez ces étapes :
    • Allez dans le menu "Team" -> "Subversion" -> "Checkout..."
    • Entrez l'URL du dépôt SVN en tant que "Repository URL"
    • Cliquez sur "Next"
    • Cliquez sur "Finish"
    • Attendez un moment
    • Un popup titré "Checkout Completed" apparait, cliquez sur "Create Project..."
    • Sélectionnez "Java Project with Existing Sources"
    • Cliquez sur "Next"
    • Entrez un nom de projet et cliquez sur "Next"
    • Dans la section "Source Package Folders", cliquez sur "Add Folder..." et sélectionnez le seul répertoire
    • Cliquez sur "Next"
    • Cliquez sur "Finish"
    • Dans l'onglet "project", sélectionnez le noeud "Libraries" -> "Add JAR/Folder" et sélectionnez tous les JARs de tous les sous-répertoires du répertoire "lib"
    • Allez dans l'onglet "project" -> "Source packages" -> "default package" -> build.xml -> build -> "Run Target" pour construire le projet
    • Allez dans l'onglet "project" -> "Source packages" -> "default package" -> build.xml -> run-game -> "Run Target" pour lancer le jeu
  • Dans les autres EDIs, débrouillez-vous, je n'ai pas de temps à perdre avec les EDIs propriétaires et ceux avec une audience confidentielle

JNDT: Java Native Deployment Toolkit

Introduction

JNDT est utilisé pour générer les paquets applicatifs natifs autonomes du projet. Il fait à partir d'un "fat JAR" des bundles faciles à déployer et à installer. Il génère à la fois des installeurs natifs pratiques avec une intégration complète dans le bureau et des archives comme solutions de repli.

C'est un empaqueteur multi-plateforme, c'est-à-dire qu'il est à la fois capable de tourner sous plusieurs systèmes d'exploitation et de générer des paquets natifs pour un système d'exploitation différent de celui sur lequel il s'exécute.

Code source

Le code source de JNDT est dans un seul fichier de bibliothèque Ant ici.

Configuration

Les dépendances de JNDT sont ici, vous devez les copier dans un répertoire et renseigner la valeur de la propriété Ant "jndtlibdir" avec ce chemin ("lib/ant" dans l'exemple ci-dessous) :

<property name="jndt-version" value="0.0.0"/>
<property name="jndt-release" value="0"/>
<property name="jndt-jar-filename" value="jndt-${jndt-version}-${jndt-release}.jar"/>
<property name="jndtlibdir" value="lib/ant"/>
<taskdef resource="jndt/antlib.xml" classpath="${jndtlibdir}/${jndt-jar-filename}"/>

Nullsoft Scriptable Install System (NSIS), disponible dans de nombreuses distributions GNU Linux, sous Microsoft Windows et sous Mac OS X, doit être installé séparément pour générer un installeur pour Microsoft Windows.

Manuel des tâches Ant

JNDT est composé des 4 tâches Ant ci-dessous:

create-app-dir
